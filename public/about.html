1DV528 Course Expectations & Learning Goals

My core expectation for the 1DV528 Web Programming course aligns directly with the goal of Part A: Developing Browser Applications. I aim to master the separation of concerns (A.2), ensuring HTML, CSS, and JavaScript are clearly distinct. Furthermore, I expect to understand the browser's architecture (A.1), learn how to store data persistently on the client (A.3), and utilize asynchronous communication and browser APIs to build optimized and accessible Progressive Web Applications (PWAs) (A.5).

The transition to Part B: Developing Server Applications is a major focus. I anticipate gaining proficiency in Node.js (B.1) and using the Express framework to manage routing and server-side data models. A key practical goal is to implement full CRUD (Create, Read, Update, Delete) functionality with persistent data storage (B.2) using an ODM. Critically, I expect to master the creation of RESTful web services (B.7) and the implementation of secure protocols, including authentication/authorization (B.3) and safeguarding against common OWASP attack vectors (B.4).

Ultimately, this course is about holistic web development. I hope to achieve the high-level goal (B.8) of being able to a complex problem, choose the right architecture and protocols, and successfully develop and deploy a complete solution. Finally, I look forward to engaging with the ethical problem-solving aspects (B.9) regarding the handling and storage of personal user data on the web.
